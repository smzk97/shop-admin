## 后台管理项目（前端）

* #### 技术栈：
  
  vue3(框架) + vite(启动) + Element-plus(组件库) + axios(请求库) + pinia(状态管理) + windicss(css框架) + vue-router(路由管理) + VueUse(轮子)

* #### 项目入口介绍
  
  ![](.\MdImg\1.png)

* #### 文件配置
  
  * `api`：用来统一管理接口请求   =>>>`axios`
  
  * `composable`：封装一些公用的方法  =>>>`Element-plus、VueUse`
  
  * `page`：可复用的单文件组件 =>>>`vue`
  
  * `router`：进行路由管理 =>>>`vue-router`
  
  * `store`：进行公有资源管理 =>>>`pinia`
  
  * `axios.js`：axios实例的创建，包括拦截器的编写
  
  * `permission.js`：全局守卫的编写

* #### 功能列表
  
  * **登录**
    
    1. 登录页面
       
       这里使用了`Element-plus`的`Layout`布局，将页面分成了两大份，使用了`flex`来管理容器排版，同时使用`windicss`替代传统`css`写法，提升了开发效率。表单使用了`Elment-plus`的组件，同时支持表单本地验证。
    
    2. 功能实现
       
       使用`axios`进行登录接口的请求，并将相应的接口请求封装在api文件下，将请求得到的token存储在cookie中，并通过拦截器设置每次请求的请求头，携带上cookie中的token；在保存用户信息时，通过在前置守卫中判断是否cookies携带token，来更新用户信息，使用`pinia`将信息保存到缓存中，同时实现了防止用户重复登录的功能；请求成功与否，使用了相应的组件实现了消息的通知；请求成功之后跳转到后台首页。
  
  * **退出登录**
    
    使用相应的组件进行消息弹框提示，点击确定进行接口请求，请求成功之后删除token，删除用户信息，并消息提示，跳转到登录页面。
  
  * **全局进度条和动态标题**
    
    使用`nprogress`包，通过拦截器或者守卫执行相应的逻辑，实现进度条的显示；在router中将标题存储在meta中，并使用守卫动态设置页面标题。
  
  * **拦截器和守卫**
    
    拦截器用来设置每次请求之前需要携带的信息，包括cookie和请求头等，同时可以用来处理loading，以及请求成功与否相应的处理；
    
    守卫一般用来进行权限验证以及标题的修改。
  
  * **后台主布局**
    
    使用element-plus的container布局容器，主要分成4个部分，分别是头部、侧边栏、标签栏、主界面，使用一个单文件组件作为父组件进行管理，封装4个子组件即可。
    
    * 头部
      
      固定写死，脱离正常文档流，主要有文字提示、菜单收缩、刷新、全屏、用户密码修改这几个交互逻辑
      
      * 文字提示
        
        使用Tooltip组件对图标进行悬停显示文字
      
      * 菜单收缩
        
        与侧边栏设计有关，更多详情后续参考
      
      * 刷新
        
        主要依托原生js事件中的location.load方法进行交互
      
      * 全屏
        
        使用VueUse中的useFullscreen Api，同时借助条件渲染，实现文字提示以及图标的动态渲染
      
      * 用户信息修改
        
        使用Dropdown下拉菜单来实现特殊功能的显示，同时使用内置的command属性监听点击事件，触发回调。
        
        退出登录需要触发相应的退出登录请求，封装在相应的api文件中，执行接口请求、删除本地用户信息、删除cookie、返回登录页、消息通知的回调。
        
        密码修改需要使用drawer抽屉组件实现相应的样式渲染，由于在其他逻辑中常用到该组件，对此将组件进行了封装。使用了defineExpose、defineProps进行组件间的数据传递，接收了 title以及size作为参数，动态渲染子组件的内容。
        
        密码修改使用了自定义的表单校验，同时也需要触发密码修改的请求。
    
    * 菜单栏
      
      也是固定写死，脱离正常的文档流，需要后端数据进行数据交互以及路由的动态添加。
      
      * 菜单收缩
        
        根据菜单栏的宽度进行状态的动态更改，初始时菜单栏宽度是固定的，点击按钮触发回调会动态改变菜单栏宽度，图标的样式也会跟随自动更改。
      
      * 数据交互    
        
        在每次进入登录成功后的页面时，返回的用户信息数据包括可显示的菜单栏数据，在每次触发相应的接口请求时都会在前置守卫将用户的信息进行缓存，使用pinia进行状态管理。可以通过列表渲染实现，同时实现刷新保持组件活跃状态，并通过回调事件实现路由跳转。
      
      * 动态路由的添加
        
        由于每个管理员的权限不同，所能访问的接口以及可视的菜单按钮不同，在原先路由中只存在公有的路由选项，包括登录、首页、404页面。因此需要在获取用户信息之后进行动态路由的添加。
        
        首先需要有一个可匹配的路由选项集合，包括路由的路径、指向的模板以及相关的元信息。可以通过递归实现相应的逻辑，同样可以在前置守卫执行该方法，需要使用vue-router中的addRoute以及hasRoute相关接口。
        
        需要注意的是，在刷新之后，由于缓存的删除，初始化时动态路由并不会存在，因此会被404捕获，这时需要重定向，让动态路由缓存到本地，可以通过判断动态路由是否新增来实现相应的逻辑。
    
    * 标签栏
      
      可以通过点击不同标签页实现页面内容的切换，类似于浏览器的标签页，同时能于左侧的菜单栏关联起来，并且实现持久化存储，在刷新之后仍能保持原先状态，还有动态删除标签页的功能。
      
      初始时固定提供首页的标签列表，点击侧边菜单栏会发生路由的转换，可以通过监听路由的变换，并与已存的标签列表作比对，判断是否添加新的标签信息，凭借vue响应式属性实现标签页的动态渲染，并将此数据储存于cookie中，实现数据持久化。
      
      可以根据当前的路由地址来实现侧边栏与标签页的绑定，实现激活页面的一致性。
      
      标签页的删除可以通过遍历已存在的标签列表，动态更改标签的激活属性以及标签列表的值，实现动态删除。相应的标签列表可以在初始时通过cookie读取。
      
      当然除了单个点击删除，还提供了“关闭其他”和“关闭全部”的功能，并固定保留首页。
    
    * 主界面
      
      实现内容的缓存
      
      * 缓存
        
        使用vue中的KeepAlive实现组件内容的缓存
